## Design note (not a verdict — informational)

`internal/refractor/consumer/manager.go` was not modified (per the dev's File
List). `cmd/refractor/main.go` still calls `manager.Add(ctx, r.ID)` to create the
durable up-front and `manager.Consumer(r.ID)` to hand the lag poller and
heartbeater's `LagProvider` a `jetstream.Consumer` handle (`main.go:144-153`,
`:317-324`). The pipeline's `RunOn`/`supervisor.Add` then idempotently
re-`CreateOrUpdateConsumer`s the same durable name. `consumer.Manager` is
therefore a live code path (not dead weight), retained as the handle source for
lag/heartbeat metrics — a "thin shim" outcome explicitly permitted by the story
("judged by the regression net, not by file count"). Both `manager_test.go` and
the new `pipeline_test.go`/e2e suites are green, so the regression net is
satisfied either way.
